# Legacy jsc harness (preserved verbatim)

These are the JavaScriptCore test suites that guarded builds 39–44 of the single-file app
(now `legacy/index.html`). They are checked in as the reference for the engine-extraction
parity work; the vitest port (Phase 2 of the frontend rebuild) must reproduce their results.

Notes for the port:
- `baseline40_env.js` freezes the clock at 2026-07-09 23:30 ET and routes fetches to
  `tests/fixtures/fix39` + `fix40`. All `DIR` constants in these files still point at the
  original session scratchpad — repoint them to `tests/fixtures/` when running.
- They `load()` a `pl_full.js` that was generated by extracting the `<script>` block from
  `legacy/index.html` (python: longest `<script>…</script>` match). It is derived, so it is
  not checked in — regenerate it from `legacy/index.html`.
- `tests/fixtures/baseline43.json` is the authoritative pick/ticket digest (build 43+44
  generation). The extracted TypeScript engine must produce an identical digest.
- `fix40/props_alt_250b0373676b10f51ed1c59c93714245.json` is a synthetic test double
  (Caesars alternate-ladder markets); everything else in `fix39`/`fix40` is real captured
  API output.
